### PR #9 - Terminal IPC Payload Hardening

Merged into `main` after a follow-up audit and README fix.

What changed:

- Added guards for terminal input, resize, and kill IPC payloads.
- Clamped terminal resize dimensions.
- Sanitized config on load, not only save.
- Changed pane title rendering to use `textContent` instead of interpolating into `innerHTML`.
- Rejected renderer-supplied shell paths while preserving safe default shell resolution from the main process.
- Added tests for terminal control payload validation, resize clamping, and renderer-supplied shell path rejection.
- Added README security-boundary notes.
- Fixed README shell-extension guidance so it no longer recommends arbitrary renderer-provided executable paths.

Impact:

- BasePlate's local terminal bridge is materially safer.
- This matters because the app intentionally runs real local developer tools through Electron IPC.

Validation:

- CI passed after the README follow-up commit.
